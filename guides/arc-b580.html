Intel Arc B580 × IPEX-LLM × Claude Code

Setup Local LLM
di Windows 11

Panduan langkah-demi-langkah untuk menjalankan AI lokal gratis

🎮 Intel Arc B580 12GB 💾 16GB DDR5 🪟 Windows 11
1
Install & Update Driver

Intel Arc B580 membutuhkan driver versi terbaru agar IPEX-LLM bisa mengakses GPU.

📥 Download Driver Intel Arc
  • Buka Intel Download Center — cari "Intel Arc & Iris Xe Graphics"
  • Download driver terbaru (minimal versi 32.0.101.6078 atau lebih baru)
  • Jalankan installer, restart komputer setelah selesai
  • Verifikasi di Device Manager → Display adapters → Intel Arc B580 Graphics
⚙️ Install Visual Studio 2022 Community

Dibutuhkan untuk kompilasi beberapa dependency IPEX-LLM.

  • Download Visual Studio 2022 Community (gratis)
  • Saat install, centang workload "Desktop development with C++"
ℹ️ Jika sudah menggunakan Intel Arc Control dan driver sudah terupdate, kamu bisa skip langsung ke checklist Visual Studio.
2
Download IPEX-LLM

Intel IPEX-LLM adalah versi Ollama yang dimodifikasi khusus untuk GPU Arc. Gunakan Portable ZIP — tidak perlu instalasi conda.

📦 Download Portable ZIP (Cara Termudah)
  • Buka halaman releases: github.com/intel/ipex-llm/releases
  • Cari file bernama ollama-ipex-llm-*.zip (versi Windows) — download versi terbaru
  • Ekstrak ke folder yang mudah diakses, contoh: C:\ipex-llm
📁 Struktur Folder
Setelah ekstrak 
C:\ipex-llm\
├── ollama.exe              # Binary Ollama versi Intel
├── start-ollama.bat        # Script untuk menjalankan server
├── ipex-llm-init.bat       # Inisialisasi environment
└── ...
⚠️ Jangan install Ollama standar dari ollama.com — versi resmi belum mendukung Intel Arc GPU. Gunakan hanya IPEX-LLM dari Intel.

4
Jalankan Ollama Server

Buka Command Prompt atau PowerShell, masuk ke folder IPEX-LLM, dan jalankan server.

▶ Terminal 1 — Jalankan Server
Command Prompt (cmd.exe) 
:: Masuk ke folder IPEX-LLM
cd C:\ipex-llm

:: Set environment agar GPU Arc terdeteksi
set ONEAPI_DEVICE_SELECTOR=level_zero:0
set SYCL_CACHE_PERSISTENT=1
set OLLAMA_INTEL_GPU=true
set ZES_ENABLE_SYSMAN=1

:: Jalankan server (BIARKAN TERMINAL INI TETAP TERBUKA)
.\ollama.exe serve
✅ Server berjalan jika muncul teks: Listening on 127.0.0.1:11434
📥 Terminal 2 — Download & Test Model
Command Prompt baru (buka tab baru) 
:: Download model (pilih sesuai pilihan di Step 3)
C:\ipex-llm\ollama.exe pull qwen2.5-coder:14b

:: Test apakah GPU dipakai (lihat output "gpu" pada info)
C:\ipex-llm\ollama.exe run qwen2.5-coder:14b --verbose
🔍 Verifikasi GPU Aktif

Saat menjalankan model dengan --verbose, pastikan ada output seperti ini:

Output yang diharapkan
✓ GPU layers: 33/33          ← semua layer di GPU ✅
Provider: Intel Arc (SYCL)    ← GPU Arc aktif ✅
VRAM usage: ~9.2 GB           ← penggunaan VRAM normal
⛔ Jika muncul CPU-only, cek kembali ONEAPI_DEVICE_SELECTOR dan pastikan driver Arc sudah terbaru.
5
Install Claude Code

Claude Code adalah coding agent Anthropic yang berjalan di terminal. Install via npm.

📦 Prasyarat — Node.js
  • Download dan install Node.js LTS (versi 18 atau lebih baru)
  • Verifikasi di terminal: node --version dan npm --version
⚡ Install Claude Code
PowerShell / cmd 
# Install Claude Code secara global
npm install -g @anthropic-ai/claude-code

# Verifikasi instalasi
claude --version
6
Hubungkan Claude Code ke Local LLM

Arahkan Claude Code untuk menggunakan Ollama lokal alih-alih Anthropic API.

🔌 Metode 1 — Per Session (Terminal)

Jalankan setiap kali ingin menggunakan local LLM:

cmd.exe 
set ANTHROPIC_BASE_URL=http://localhost:11434
set ANTHROPIC_AUTH_TOKEN=ollama
set ANTHROPIC_API_KEY=

claude --model qwen2.5-coder:14b
⚙️ Metode 2 — Permanen (settings.json)

Edit file %USERPROFILE%\.claude\settings.json agar otomatis setiap kali:

~\.claude\settings.json 
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:11434",
    "ANTHROPIC_AUTH_TOKEN": "ollama",
    "ANTHROPIC_API_KEY": ""
  },
  "hasCompletedOnboarding": true,
  "primaryApiKey": "sk-dummy-key"
}
🔄 Kembali ke Anthropic API

Jika ingin pakai Claude resmi lagi:

cmd.exe 
set ANTHROPIC_BASE_URL=
set ANTHROPIC_AUTH_TOKEN=
:: Atau hapus env dari settings.json
⚠️ Pastikan server Ollama (Terminal 1 dari Step 4) masih berjalan sebelum menjalankan claude.
7
Optimasi Performa

Beberapa setting tambahan untuk memaksimalkan performa B580 kamu.

🚀 Context Window — Wajib Diset!

Claude Code sangat boros context. Set minimal 32K, idealnya 64K:

Buat file Modelfile 
# Buat file: C:\ipex-llm\Modelfile
FROM qwen2.5-coder:14b
PARAMETER num_ctx 32768

# Kemudian buat model custom:
C:\ipex-llm\ollama.exe create mymodel -f C:\ipex-llm\Modelfile
C:\ipex-llm\ollama.exe run mymodel
⚡ Environment Variables Tambahan
Tambahkan ke start script 
:: Nonaktifkan KV Cache invalidation (performa +90%)
set CLAUDE_CODE_ATTRIBUTION_HEADER=0

:: Matikan telemetri
set DISABLE_TELEMETRY=1

:: Nonaktifkan prompt caching (local model tidak support)
set DISABLE_PROMPT_CACHING=1
📊 Ekspektasi Performa B580
qwen2.5-coder:7b ~40–60 tok/s
qwen2.5-coder:14b ~20–35 tok/s
qwen3:14b ~20–30 tok/s
